# 第一阶段：指针的指尖艺术 —— 链表技巧 (Linked List)

如果说数组是排好队的“固定座位”，链表就是玩“老鹰捉小鸡”的小朋友，每个人只知道自己后面是谁，但不知道前面是谁。

## 1. 核心思维：“链表手术”模型

链表题最难的地方在于：**当你断开一个指针时，如果你不提前找个临时工（变量）拉住后半段，后半段的人就会全部“迷失”在内存里。**

所以，处理链表就像在给人做血管缝合手术：
- **镊子（指针）**：`pre`, `cur`, `next`。
- **缝合顺序**：必须先接好前面的，再接后面的，顺序一旦乱了，整条链条就断了。

## 2. 三大核心神技

### A. 哑节点 (Dummy Node) —— 永远的后盾
这是链表题最重要的小技巧。它是一个人为创造出来的节点，放在真正的 `head` 之前。
- **为什么要用？**：当你需要删除头节点、或者从头重新拼接链表时，操作 `head` 会非常痛苦。
- **口诀**：`dummy = ListNode(0, head)`。这样，真正的答案永远在 `dummy.next`。

### B. 双指针/三指针平移 —— 协同工作
在链表中，我们很少让一个指针落单。
- **思维**：用 `pre` 指向已处理部分的末尾，用 `cur` 指向正在处理的节点，用 `tmp` 甚至是 `nxt` 指向还没处理的地方防止丢失。

### C. 断链与重组 —— 指针重定向
典型的操作如反转链表：
1. 先记住 `cur.next`。
2. 把 `cur.next` 的指向反转（指向 `pre`）。
3. 整体向后平移。

## 3. 大脑模拟：反转链表 (Reverse Linked List)

假设链表是 `1 -> 2 -> 3 -> None`

1. **初始**：`pre = None`, `cur = 1`。
2. **第一步（救命）**：拿个变量 `nxt` 拽住 `2`（即 `nxt = cur.next`）。
3. **第二步（翻转）**：让 `1` 的手指向 `pre`（即 `cur.next = pre`）。此时变成了 `1 -> None`。
4. **第三步（接力）**：`pre` 变成 `1`，`cur` 变成 `nxt` (也就是之前的 `2`)。
5. **循环**：直到 `cur` 变成 `None`。

### D. “尺子定位法” —— 处理倒数节点
处理“倒数第 N 个”问题时，链表无法倒流是最大痛点。
- **思维工具（尺子）**：
  1. 拿一把长度为 N 的虚拟尺子（即快慢指针，让 `fast` 先跑 `N` 步）。
  2. 保持尺子长度，拖着它匀速向右滑。
  3. 当右尖端（`fast`）触底时，左尖端（`slow`）指的地方就是我们要找的黄金位置。
- **练习推荐**：19. 删除链表的倒数第 N 个节点、61. 旋转链表

## 5. 模式应用与实战意义

| 技巧/题目 | 核心思维模型 | 可以解决哪类问题？ |
| :--- | :--- | :--- |
| **哑节点 (Dummy)** | **哨兵保护** | 凡是涉及到“头节点可能被删除/修改”或“从空开始构建链表”的场景。 |
| **206. 反转链表** | **原地手术** (逐个重连) | 需要改变链表物理指向，或者是区间内局部翻转。 |
| **19. 删除倒数第 N 个** | **尺子定位** (前后联动) | 在单向流动的链表中，精确定位距离末尾特定距离的位置。 |

## 6. 关键避坑指南

- **不要在循环中使用已经断开的指针**：比如你已经写了 `cur.next = pre`，那么在这一行之后，原来的 `cur.next` 就再也找不回来了。
- **判断 `None` 的深度**：除了判断 `while cur`，有时候还要判断 `while cur.next`。多一层判断，多一层保险。
