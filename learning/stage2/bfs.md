# 第二阶段：水波的无差别扩散 —— 广度优先搜索 (BFS)

如果说深度优先搜索（DFS / 递归）像是一个执着的探险家，一条路走到黑，撞了南墙才回头；那么广度优先搜索（BFS）就像是一颗投入水中的石子，以自己为圆心，向外一圈一圈均匀地荡开水波。

## 1. 核心思维：“排队打饭”与“代沟绝对隔离”

BFS 的核心不在于树或者图，而在于**队列 (Queue)** 这个数据结构。队列的特性是“先进先出 (FIFO)”。

**大脑模拟**：
1. **第一圈波纹**（根节点）先进入队列。
2. 只要队列不空，我们就开始营业：把排在队首的人叫出来（出队）。
3. 这个人处理完自己的事后，必须把他的**直系下属（左右孩子）**拉过来，排到队伍的**最末尾**。
4. 循环往复。

由于队列“先来后到”的刚性铁律，**第一层的所有人绝对会在第二层的第一人之前被处理完**。这种“绝对的代际隔离”，让 BFS 成为解决一类特定问题的终极杀器。

## 2. 模式识别：什么时候必须用 BFS？

只要你在题目中看到了以下字眼，毫不犹豫，立刻默写 BFS 模板：
- **层序遍历**（“一层一层地输出”、“每一层的最大/小值”、“右视图”）
- **最短路径**（“最少操作次数”、“从起点到终点的最短距离”、无权图求最短路）

> [!TIP]
> 为什么 BFS 能求最短路径？因为波纹是均匀扩散的！当某一层的水波第一次触碰到目标地点时，它所经历的圈数（层数），在物理规律上**注定是**最短的。DFS 可能会绕极其远的路才瞎猫撞上死耗子，但 BFS 走到哪，哪里就是最优解。

## 3. 标准解题模板：两层循环的“钢铁洪流”

写 BFS 时，千万不要嵌套递归！它需要的是**一个外部的 `while` 控制全局，一个内部的 `for` 控制层级**。

```python
from collections import deque

def bfs_template(root):
    if not root:
        return []
    
    # 1. 队列初始化：起点的波纹放入队列
    queue = deque([root])
    res = []
    
    # 2. 只要水波还在荡漾
    while queue:
        level_size = len(queue) # 【极其关键！】在此刻锁死当前层的“波纹数量”
        level_nodes = []        # 存放这一层的处理结果
        
        # 3. 把这一层的所有波纹横扫一遍
        for _ in range(level_size):
            node = queue.popleft() # 出队
            level_nodes.append(node.val)
            
            # 把这个波纹激发出的下一层波纹（左右下属），排到队尾去！
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
                
        # 这一层横扫完了，统一归档
        res.append(level_nodes)
        
    return res
```

### 为什么要有 `level_size = len(queue)` 这一步？
因为队伍是**动态**的！在遍历这一层的人时，他们的小弟正在源源不断地排到队尾去。
如果你不去在循环一开始“截断”当前层的总人数（`level_size`），你会连着把第二层、第三层提前处理掉，导致“分层”溃败！这是 BFS 里最经典的新手死法。
