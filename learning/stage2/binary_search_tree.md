# 第二阶段：规则带来的降维打击 —— 二叉搜索树 (BST)

在普通的二叉树中，如果你要找一个特定的值，你只能像无头苍蝇一样把所有节点都遍历一遍（$O(N)$）。
但是，二叉搜索树（Binary Search Tree，简称 BST）给自己定下了一套极其严格的“家规”：

**对于树上的任意一个节点：**
1. **它的左子树里所有的节点值，都比它小。**
2. **它的右子树里所有的节点值，都比它大。**

## 1. 核心思维：“自带导航”的树

因为有了这套家规，BST 就像是一棵“自带导航系统”的二叉树。它把我们在**二分搜索**中学到的“一刀两断”的排除法，完美地融入到了树的结构中。

- 目标值比当前节点小？**左边！右子树连看都不用看了！**
- 目标值比当前节点大？**右边！左子树可以直接抛弃！**
- 目标值等于当前节点？**找到了！**

在 BST 中，每一次向下走一步，你都能排除掉一半（或者一部分）的错误答案。这就是为什么它的搜索效率可以达到惊人的 $O(\log N)$（在树比较平衡的情况下）。

## 2. 模式识别

只要题目中出现了 **“二叉搜索树”** 或者 **“BST”** 这几个字，你的第一反应就应该是：
🌟 **利用有序性进行排除！我绝对不要遍历整棵树！**

## 3. 标准解题模板

BST 的搜索逻辑简直就是把人的直觉翻译成了代码：

```python
def search_bst(root, target):
    # 终止条件：走到空了，或者找到了
    if not root:
        return None
    if root.val == target:
        return root
        
    # 核心判断逻辑：利用规范进行分支选择
    if target < root.val:
        return search_bst(root.left, target) # 去左边找，不用加其他逻辑了
    else:
        return search_bst(root.right, target) # 去右边找，不用加其他逻辑了
```

仔细观察这段代码，你会发现它和普通的二叉树递归有一个极大的不同：
普通二叉树（比如求深度），你需要等左右子树都把结果汇报上来，再去汇总（`1 + max(left, right)`）。
而 BST 呢？它是一种**单边探索**！你做完了判断，直接把结果交给选中的那个子树去办就行了，甚至不用等它回报给你进行加工。直接 `return` 它的结果。

## 4. 进阶思维：“区间约束下发” (携带尚方宝剑查岗)

BST 的“左小右大”不仅仅是和直接相连的父节点比，而是和**所有的祖先节点**比！

**警惕经典陷阱**：
很多初学者写 `isValidBST` 时，只检查 `root.left.val < root.val` 和 `root.right.val > root.val`。
此时如果右子树里藏着一个极小的值（比如比根节点还要小），这个只顾眼前的瞎子反而是查不出来的！

**解题模型**：
不要只往左右走，要**带着区间的上下限（尚方宝剑）**往左右走：
1. 向左走时，上限被当前节点卡死了（最大不能超过当前节点）。
2. 向右走时，下限被当前节点垫高了（最小不能低于当前节点）。
3. 每个子节点被访问时，都要先看看自己有没有越俎代庖（超出上下限）。

## 6. 学术语对照：大纲里的关键词都在哪？

虽然大纲中并没有将 BST 单列为一个大类别，但它完美契合了以下概念：

| 大纲/通用关键词 | 我们的实战模型 | 对应你代码中的动作 |
| :--- | :--- | :--- |
| **有序结构** | **左小右大** | 比较 `target` 和 `root.val` 的大小关系。 |
| **二分思维** | **单边探索 (剪枝)** | 只要进入了左子树，就相当于瞬间把右子树所在的分支全部排除了。 |
| **状态传递** | **区间查岗** | 将 `(min_val, max_val)` 作为参数，在递归调用时不断缩窄并向下传递。 |
